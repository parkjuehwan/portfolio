Background

분석 배경: 농작물 피해 예방

02 / 11

문제 정의

  • 늦봄과 초가을의 갑작스러운 서리는 과수나 노지 채소에 큰 피해를 줄 수 있습니다.
  • 거시적 기상 예보만으로는 농경지 단위의 미세 기후 변화를 반영하기 어렵습니다.
  • 기상 관측 데이터를 활용해 특정 지역의 서리 발생 여부를 예측하는 모델이 필요합니다.

분석 가치

  • 농작물 피해 예방을 통한 농가 소득 보호
  • 방재 인력·장비 등 농업 자원의 효율적 배분
  • 기후 변화에 따른 이상기후 대응력 향상

AutoML 시연과 Python 분석을 함께 정리

03 / 11

초급 자료: AutoML 활용

  • 코딩 없이 데이터 업로드 → 모델링 실행 → 결과 확인 흐름
  • 분류 모델 선택 후 타깃 컬럼을 FROST로 지정
  • 모델별 성능, ROC Curve, 변수 중요도 등 대시보드 기반 결과 해석

중급 자료: 코드 기반 분석

  • 결측치 처리, 파생 변수 생성, 스케일링, 언더샘플링 수행
  • Logistic Regression, Random Forest, Gradient Boosting, Extra Trees 비교
  • 혼동 행렬과 Recall/F1/ROC-AUC 중심으로 성능 해석

포트폴리오 관점

단순히 AutoML 결과를 확인한 것이 아니라, 불균형 데이터 문제와 평가지표 선택 이유를 설명할 수 있도록 분석 흐름을 구조화했습니다.

서리 예측에서 중요한 기준

농작물 피해와 직접 연결되는 것은 실제 서리를 놓치는 FN이므로, Recall을 높이면서도 오경보 비용을 줄이는 Precision/F1 균형이 중요합니다.

수행 역량과 직무 연결점

11 / 11
1. 불균형 데이터 문제 정의서리 발생 비율이 낮은 데이터에서 Accuracy만으로는 부족하다는 점을 설명하고, Recall/F1 중심의 평가 기준을 정리했습니다.
2. 전처리와 샘플링 전략 이해결측치 처리, 스케일링, 언더샘플링 등 모델 학습 전 데이터 품질을 개선하는 과정을 구조화했습니다.
3. 모델 비교와 결과 해석4개 분류 모델을 동일 기준으로 비교하고, 최종 모델을 Gradient Boosting으로 선정한 이유를 성능 지표로 설명했습니다.
4. 제조·설비 데이터로의 확장성희귀 이벤트 탐지, 이상 징후 분류, 고장 발생 예측처럼 불균형 데이터가 많은 현업 문제에 연결할 수 있습니다.
